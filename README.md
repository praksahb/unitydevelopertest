# Unity Game Developer Task

## Objective:

The objective of this task is to create a Gravity Manipulation Puzzle game using Unity. The player's goal is to collect all the cubes by manipulating gravity. The task involves cloning a base project provided with assets, characters, animations, and materials. The game should include character movement, gravity manipulation, camera setup, animations, a timer, and game over conditions. The final deliverables include a well-written and clean codebase, a shared project on Git, and standalone builds for Windows and Mac.

## Base Project:

Clone the base project: https://gitlab.com/chinmaygawande/unitydevelopertest.git

## **Deliverables:**

1. Character Movement:
    - Implement character movement using the W, A, S, and D keys.
    - Implement jumping functionality using the Space key.
2. Gravity Manipulation:
    - Implement gravity manipulation using arrow keys.
    - Show a hologram of the character's reference direction when arrow keys are pressed.
    - When the Enter key is pressed, manipulate the gravity in the selected direction.
    - <u>***Note: You can use any logic to achieve the gravity manipulation effect; using Rigidbody is not mandatory.***</u>
3. Camera Setup:
    - Implement a third-person camera setup for the game.
4. Animations:
    - Properly implement character animations provided in the base project.
5. Timer:
    - Display a timer on the screen with a limit of 2 minutes.
    - If the player fails to collect all the cubes within the time limit, the game should be over.
6. Game Over Conditions:
    - If the player falls freely (no contact with any surface), the game should be over.
7. Code Quality:
    - Write well-structured, clean, and optimized code.
    - Use proper code documentation and follow coding best practices.
8. Git Repository:
    - Share the complete project on Git by creating a repository.
    - Commit code changes regularly with meaningful commit messages.
9. Standalone Builds:
    - Generate standalone builds of the game for both Windows and Mac platforms.
    - Ensure that the builds are functional and can be executed without additional dependencies.

## **Rules:**

1. Use the provided base project for assets, characters, animations, and materials. Do not waste time downloading or creating additional models or characters.
2. The character should move using the W, A, S, and D keys, jump using the Space key, and manipulate gravity using the arrow keys.
3. The camera setup should be third person, providing a suitable view of the character and the environment.
4. Implement the timer functionality, displaying a 2-minute time limit on the screen. If the player fails to collect all the cubes within the time limit, the game should end.
5. Implement game over conditions: if the player falls freely without any surface contact, the game should end.
6. Write clean and well-structured code, following coding best practices. Properly document the code to enhance readability.
7. Share the complete project on Git, committing code changes regularly with meaningful commit messages.
8. Generate standalone builds of the game for both Windows and Mac platforms, ensuring they are functional and can be executed independently.
